# 浏览器扩展优化 - 账号切换检测

## 🎯 解决的问题

### 原问题
用户在浏览器中切换抖音账号后：
1. Cookie 已经变成新账号的了
2. 但页面的 DOM 和 localStorage 还是旧账号的数据（昵称、头像、sec_uid）
3. 点击"重新抓取"后，会得到：
   - ✅ 新账号的 Cookie
   - ❌ 旧账号的昵称和头像
4. 导入到后台后，**账号信息错乱**

### 用户体验问题
- 用户看到的昵称/头像与实际 Cookie 不匹配
- 导入后创建的账号信息是错的
- 需要手动编辑或删除重新导入

## ✅ 优化方案

### 1. 自动检测账号切换
通过对比 Cookie 的 sessionid 指纹（最后8位）：
- **第一次抓取**：记录 sessionid 指纹
- **再次抓取**：对比新旧指纹
  - 指纹相同 → 同一账号，正常抓取
  - 指纹不同 → 账号切换了，检查页面是否刷新

### 2. 强制刷新提示
如果检测到账号切换但页面未刷新：
```
⚠️ 检测到账号切换
新 Cookie 指纹：…abc12345
旧 Cookie 指纹：…xyz67890

请刷新当前页面（F5 或 Cmd+R）后再重新抓取
否则会获取到混乱的账号信息（新 Cookie + 旧账号昵称/头像）
```

### 3. 醒目的警告样式
- 黄色警告框，高亮显示
- Cookie 指纹用红色背景标注
- 明确提示操作步骤

### 4. 更新底部说明
在页面底部用醒目的橙色文字强调：
```
⚠️ 切换账号后必须刷新页面

切换抖音账号后，请在新账号页面按 F5 或 Cmd+R 刷新页面，
再点「重新抓取」。核对上方昵称、头像、账号指纹是否已变化，
确保 Cookie 和账号信息匹配。
```

## 🔍 技术细节

### Cookie 指纹提取
```javascript
function accountFingerprint(cookie) {
  const m = /(?:^|;\s*)sessionid(?:_ss)?=([^;]+)/.exec(cookie || '');
  if (!m) return '';
  const v = m[1].trim();
  return v.length <= 8 ? v : `…${v.slice(-8)}`;
}
```

### 账号切换检测逻辑
```javascript
// 1. 获取当前 Cookie 指纹
const fp = accountFingerprint(cookie);

// 2. 与上次对比
if (lastFingerprint && fp && lastFingerprint !== fp) {
  // 3. 检查页面信息是否也更新了
  const pageChanged = ls.account && ls.account.nickname;
  if (!pageChanged) {
    // Cookie 变了但页面没变 → 页面未刷新 → 显示警告并阻止
    显示警告();
    return;
  }
}

// 4. 记录当前指纹
lastFingerprint = fp;
```

### 为什么要检测
- **Cookie 是即时的**：`chrome.cookies.getAll()` 立即返回当前浏览器的 Cookie
- **页面内容是延迟的**：DOM、localStorage、`__INITIAL_STATE__` 需要页面刷新才会更新
- **时间差导致混乱**：用户切号后没刷新，插件会拿到"新 Cookie + 旧页面信息"

## 💡 用户指南

### 正确使用流程
1. **首次抓取**：打开扩展，自动抓取当前账号
2. **切换账号**：
   - 在抖音页面切换到新账号
   - **按 F5 或 Cmd+R 刷新页面**
   - 打开扩展或点击"重新抓取"
   - 核对昵称、头像、指纹是否已变化
3. **复制导入串**：点击"复制一键导入串"
4. **导入后台**：粘贴到 DyAuthReply 后台"新增账号"

### 如果忘记刷新
扩展会自动检测并显示醒目警告：
```
⚠️ 检测到账号切换
请刷新当前页面（F5 或 Cmd+R）后再重新抓取
```
