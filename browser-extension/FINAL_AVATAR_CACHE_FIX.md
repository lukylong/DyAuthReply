# ✅ 浏览器扩展终极优化 - 隐藏不可信头像

## 🎯 核心问题

**浏览器图片缓存无法破坏**：即使添加时间戳、硬刷新，浏览器仍然显示缓存的旧头像。

### 根本原因
- 浏览器对图片的缓存策略非常激进
- CDN 缓存策略也会影响
- 添加时间戳不一定生效（可能被 CDN 忽略）
- 用户无法仅通过头像判断账号是否正确

## ✅ 终极解决方案

### 1. 直接隐藏头像（切换账号时）
```javascript
// 如果检测到账号切换，隐藏头像
if (isSwitched) {
  avatarEl.hidden = true;
} else if (acc.avatar) {
  // 正常显示
  avatarEl.src = avatarUrl;
  avatarEl.hidden = false;
}
```

**理由**：
- ❌ 显示旧头像 → 误导用户
- ✅ 隐藏头像 → 用户知道需要核对其他信息

### 2. 强调核对昵称和抖音号
新的警告提示：
```
⚠️ 检测到账号切换
Cookie 指纹已变化：…xyz67890 → …40e54e5c
当前识别昵称：哈啰

注意：头像已隐藏（浏览器缓存无法清除旧头像）

请核对以下信息是否正确：
1. 上方显示的昵称是否是新账号的？
2. 上方显示的抖音号是否是新账号的？
3. 去抖音页面本身查看个人中心，核对昵称

如果信息不正确，请：
• 硬刷新（Ctrl+Shift+R / Cmd+Shift+R）
• 或关闭标签页重新打开新账号

核对无误后再复制导入串！
```

### 3. 用户体验改进

#### 切换前（正常抓取）
```
🔵 [头像]  哈啰
抖音号 165112170 · 指纹 …40e54e5c
```

#### 切换后（检测到切换）
```
[无头像]  哈啰 ⚠️
抖音号 165112170 · 指纹 …40e54e5c

[黄色警告框]
⚠️ 检测到账号切换
头像已隐藏...
请核对昵称和抖音号...
```

## 📊 对比方案

| 方案 | 效果 | 问题 |
|------|------|------|
| 添加时间戳 | ❌ 无效 | 浏览器/CDN 仍然缓存 |
| 硬刷新提示 | ⚠️ 部分有效 | 用户不一定会做 |
| **隐藏头像** | ✅ **有效** | 用户知道需要核对其他信息 |

## 🎯 新的用户流程

### 切换账号操作
1. 在抖音页面切换账号
2. 刷新页面（F5 或 Ctrl+Shift+R）
3. 打开扩展
4. 看到：
   - ⚠️ 黄色警告框
   - **头像被隐藏了**（不再显示旧头像）
   - 昵称后有 ⚠️ 图标
5. **核对信息**：
   - ✅ 昵称是新账号的吗？
   - ✅ 抖音号是新账号的吗？
   - ✅ 去抖音页面本身核对
6. 核对无误 → 复制导入串

### 为什么这样更好

#### 之前（显示旧头像）
- 用户看到旧头像 → 认为是旧账号
- 但昵称是新的 → 混乱
- 不知道相信哪个

#### 现在（隐藏头像）
- 没有头像 → 用户知道需要核对
- 专注于昵称和抖音号
- 明确的核对清单

## 📝 代码修改

### popup.js

1. **renderAccount 函数**
```javascript
// 检测到切换时隐藏头像
if (isSwitched) {
  avatarEl.hidden = true;
} else if (acc.avatar) {
  // 正常显示（带时间戳）
  avatarEl.src = avatarUrl + '?_t=' + Date.now();
  avatarEl.hidden = false;
}
```
