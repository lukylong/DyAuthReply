# 🔧 浏览器扩展增强版 - 更强的账号切换检测

## 🎯 发现的新问题

用户反馈：即使刷新了页面，抓取到的**头像仍然是旧账号的**。

### 根本原因
1. **浏览器图片缓存**：即使页面刷新了，浏览器仍然从缓存中加载头像图片
2. **页面缓存**：有时候 F5 刷新不够彻底，DOM 和 localStorage 仍有旧数据
3. **扩展 popup 缓存**：popup 关闭再打开时，上一次的数据可能残留

## ✅ 增强方案

### 1. 头像 URL 添加缓存破坏参数
```javascript
// 之前
avatarEl.src = acc.avatar;

// 现在：添加时间戳，强制浏览器重新加载
const avatarUrl = acc.avatar.includes('?')
  ? `${acc.avatar}&_t=${Date.now()}`
  : `${acc.avatar}?_t=${Date.now()}`;
avatarEl.src = avatarUrl;
```

### 2. 昵称添加警告标记
当检测到账号切换时，在昵称后显示 ⚠️ 图标：
```
哈啰 ⚠️
```
鼠标悬停显示："检测到账号切换，请核对信息"

### 3. 更明确的警告提示
```
⚠️ 检测到账号切换
Cookie 指纹已变化：…xyz67890 → …40e54e5c
当前识别昵称：哈啰

请仔细核对上方昵称和头像是否正确。
如不正确，说明页面缓存未清除，请：
1. 再次刷新页面（F5）
2. 或硬刷新（Ctrl+Shift+R / Cmd+Shift+R）清除缓存
3. 或关闭标签页重新打开新账号页面
4. 核对无误后再复制导入串
```

### 4. 记录昵称用于双重检测
- 之前只记录 sessionid 指纹
- 现在同时记录昵称
- 未来可以对比昵称是否也变化了

## 🎯 解决方案总结

### 问题分层
1. **Cookie 层面**：已经切换（sessionid 指纹变了）✅
2. **页面内容层面**：可能有缓存（昵称/DOM 数据）⚠️
3. **图片缓存层面**：浏览器缓存头像 ❌

### 解决策略
1. **检测**：对比 sessionid 指纹，检测账号切换
2. **提示**：醒目的黄色警告框 + 昵称警告图标
3. **破坏缓存**：头像 URL 添加时间戳
4. **用户教育**：明确告知硬刷新（Ctrl+Shift+R）的存在

## 🔍 为什么头像会缓存

### 浏览器缓存策略
- 图片 URL 相同 → 浏览器认为是同一张图片 → 从缓存加载
- 抖音的头像 URL 通常很长时间不变
- F5 刷新**不会**清除图片缓存

### 解决方法
1. **添加时间戳**（已实现）：`?_t=1234567890`
2. **硬刷新**（提示用户）：Ctrl+Shift+R / Cmd+Shift+R
3. **关闭标签页重开**（最彻底）
